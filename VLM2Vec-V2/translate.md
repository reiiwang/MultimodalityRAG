# Abstract

多模態嵌入模型在語義相似度、資訊檢索和跨模態聚類等各種下游任務中發揮了關鍵作用。然而，現有的多模態嵌入模型（如 VLM2Vec、E5-V、GME）主要聚焦於自然圖像，對影片和視覺文件等其他視覺形式的支援十分有限。這限制了其在現實場景中的適用性，包括 AI 代理、多模態搜尋與推薦，以及檢索增強生成（RAG）等應用。

為填補這一空缺，我們提出 VLM2Vec-V2——一個用於學習多種視覺形式嵌入表示的統一框架。首先，我們推出 MMEB-V2，這是一個在 MMEB 基礎上擴展的綜合性基準測試，新增了五種任務類型：視覺文件檢索、影片檢索、時序定位、影片分類和影片問答，涵蓋文字、圖像、影片和視覺文件等多種輸入形式。其次，我們訓練了 VLM2Vec-V2——一個支援文字、圖像、影片和視覺文件輸入的通用嵌入模型。大量實驗表明，VLM2Vec-V2 不僅在新引入的影片與文件檢索任務上表現優異，在原有圖像基準測試上也超越了先前的基線模型。

透過深入評估，本研究揭示了多種多模態嵌入模型的泛化能力，並總結了統一嵌入學習的有效策略，為研究領域和實際應用中更具擴展性與適應性的表示學習奠定了基礎。

# Introduction

嵌入模型在跨模態資料連結中扮演著關鍵角色。透過將異質多模態資料編碼至共享的稠密表示空間，它們得以支援分類、聚類、檢索等眾多下游應用。近年來，嵌入模型取得了顯著進展，這在很大程度上得益於大型基礎模型的持續突破。

例如，近期文字嵌入領域的重要進展（Su et al., 2023；Wang et al., 2024a；Meng et al., 2024；BehnamGhader et al., 2024）正是透過將預訓練大型語言模型與多任務指令嵌入微調相結合而實現的。類似地，Jiang et al. (2024)、Zhang et al. (2024b) 及 Chen et al. (2025) 透過對視覺語言模型（VLMs）進行指令微調，將其轉化為高效的嵌入模型，在多項文字—圖像任務中展現出卓越性能。

現有的多模態嵌入模型主要在 MMEB（Jiang et al., 2024）和 M-BEIR（Wei et al., 2023）等資料集上訓練，這些資料集以自然圖像或照片為主，來源包括 MSCOCO（Lin et al., 2014）、Flickr（Plummer et al., 2015）和 ImageNet（Deng et al., 2009）等資料集。這些資料集未能涵蓋更廣泛的視覺資訊形式，如文件、PDF、網頁、影片、簡報等。覆蓋範圍的不足導致現有嵌入模型在許多現實任務中表現欠佳，例如文章搜尋、網頁搜尋、YouTube 影片搜尋等。

為解決上述局限，我們推出 MMEB-V2——一個先進的多模態嵌入資料集，旨在針對三種核心視覺模態（圖像、影片和視覺文件）對嵌入模型進行訓練與評估。在原有 MMEB（Jiang et al., 2024）框架的基礎上，MMEB-V2 將評估範圍擴展至五項新任務，包括四項影片相關任務——影片檢索、片段檢索、影片分類和影片問答——以及一項以視覺文件為核心的任務：視覺文件檢索。這一完整的任務體系能夠在靜態、時序和結構化視覺資料等不同場景下，對多模態嵌入模型進行全面評估。

在 MMEB-V2 的基礎上，我們進一步提出 VLM2Vec-V2——一個從最先進視覺語言模型（Wang et al., 2024b）微調而來的強大多模態嵌入模型。VLM2Vec-V2 透過混合多任務類別的指令跟隨任務進行訓練，能夠為多種視覺模態生成統一的表示。

透過 VLM2Vec-V2 與 MMEB-V2，我們旨在探討以下研究問題：多模態嵌入模型能否跨越多種視覺模態實現良好的泛化？訓練穩健且多功能的多模態嵌入模型的關鍵要素是什麼？在影片時序資訊和視覺文件結構化資訊的表示方面，主要挑戰有哪些？

本研究的貢獻體現在三個方面：

- 我們提出 MMEB-V2，這是一個用於系統評估嵌入模型在影片、圖像和視覺文件等多樣任務上的綜合資料集。
- 我們開發了 VLM2Vec-V2，一個支援多種輸入格式的統一多模態嵌入模型，能夠依據任務指令生成通用嵌入表示，以支援各類下游任務。
- 實驗結果表明，VLM2Vec-V2 在 78 個資料集上均優於先前的基線模型。透過詳細的消融實驗，我們總結出跨模態嵌入模型學習的有效訓練策略。

# MMEB-V2: Expanding Embedding Benchmarks Beyond Image-Text

我們推出 MMEB-V2，這是一個綜合性資料集，旨在評估模型在涉及文字、圖像、影片和視覺文件等多種模態組合的多模態嵌入任務上的表現。除了涵蓋自然圖像與文字的五個任務類別外，MMEB-V2 還新增了四項影片理解任務和一項視覺文件理解任務。圖 1 呈現了 MMEB-V2 的整體概覽。

每項任務向模型提供一個查詢及對應的候選回應集，目標是選出正確答案。查詢由指令、文字部分以及影片或文件組合而成。對於影片類任務，影片以從原始素材中按均勻間隔取樣的幀序列表示，以確保一致的時序覆蓋。指令用於說明模型的目標（例如：「辨識影片內容的類別。」），查詢文字可以是針對影片的問題、描述或指令（例如：「這部影片結尾時壁爐上方有幾隻紅色襪子？」、「選出包含海豚的影片片段。」）。每項任務對應特定的目標類型，依任務性質而有所不同。例如，在影片分類任務中，模型須識別活動或物體的類別標籤（如「瑜伽」或「汽車」）。

為使資料集便於未來研究使用，我們對部分資料集進行了選擇性降採樣，以確保完整基準測試能在合理時間內完成。資料集涵蓋多元領域，包括運動、物體辨識、日常生活活動，以及電影或電視節目片段，樣本來源涵蓋 YouTube、專業製作內容及群眾外包內容，兼顧多樣性與現實相關性。各任務的統計摘要見表 1，各資料集的構建細節詳見附錄 A.2。

- **影片檢索（V-RET）**：查詢由指令、與影片內容相關的描述文字及影片幀序列組成。模型須從數千部候選影片中檢索出正確對應的影片。

- **片段檢索（M-RET）**：查詢由指令、文字描述及（可選的）完整影片組成，目標是檢索出與描述最匹配的時間段。模型須從完整影片中約 10 個候選片段中選出正確片段。

- **影片分類（V-CLS）**：給定指令和影片幀序列，模型須從一組可能的類別中預測正確的類別標籤，涉及場景或動作內容。

- **影片問答（V-QA）**：輸入由指令、文字問題和影片組成。模型須從多個選項（包含一個正確答案及多個干擾項）中選出正確答案。

- **視覺文件檢索（VisDoc）**：此任務類別評估模型根據自然語言查詢檢索結構化視覺文件（如多頁 PDF 和簡報）的能力。本基準共納入五個資料集：ViDoRe V1 & V2（Faysse et al., 2024；Macé et al., 2025）和 VisRAG（Yu et al., 2024）由多個文件問答資料集組成，涵蓋廣泛的文件類型與使用場景（如圖表和簡報），但原本並非為檢索任務而設計。為加以補充，我們納入了 ViDoSeek（Wang et al., 2025）和 MMLongBench-Doc（Ma et al., 2024），這兩者提供了適合檢索評估的細粒度頁面級標註。此外，我們對這兩個資料集進行了重新格式化，以同時支援文件級與頁面級評估。VisDoc 最終分數以 24 項任務的 NDCG@5 平均值計算。

# Unified Embedding Model for Video, Image, and Visual Document

由於不同模態與任務具有各異的結構特徵與語義特性，統一跨模態與跨任務的嵌入學習本身極具挑戰性。我們的目標是將不同模態的資料對齊至共享嵌入空間，同時透過定義各任務的自然語言指令來引導模型行為。

本節概述我們的方法，包括多模態輸入格式化（第 3.1 節）、基於共享骨幹網路的統一編碼（第 3.2 節）、指令引導的對比訓練（第 3.3 節），以及用於平衡資料來源的策略性採樣（第 3.4 節）。

**3.1 多模態資料的統一表示**

我們的目標是學習一個支援多種視覺模態與任務的統一嵌入空間。這要求模型骨幹能夠靈活編碼文字、圖像與影片的交錯序列，同時處理完整影片和多頁視覺文件等長篇輸入。視覺語言模型（Liu et al., 2024a）在各項基準測試中表現優異，並已被證明是多模態嵌入模型的有效基礎（Jiang et al., 2024；Lin et al., 2024）。

基於上述要求，我們採用 Qwen2-VL（Wang et al., 2024b）作為 VLM2Vec-V2 的骨幹網路。Qwen2-VL 尤為契合我們的需求，具備以下優勢：（1）原生動態解析度，可高效處理不同解析度的輸入；（2）多模態旋轉位置嵌入（M-RoPE），能夠捕捉空間與時序結構；（3）整合 2D 與 3D 卷積的統一架構，實現一致的圖像與影片理解。這些能力使得跨異質多模態資料的可擴展性與泛化性編碼成為可能。

**3.2 對比學習**

我們採用對比訓練將視覺語言模型轉化為嵌入模型。給定一個預訓練 VLM，我們將查詢和目標分別輸入模型，透過取最後一層最後一個 token 的向量表示，得到查詢嵌入（$h_q^{inst}$）和目標嵌入（$h_{t^+}$）。為訓練嵌入模型，我們採用標準 InfoNCE 損失（Oord et al., 2018）$\mathcal{L}$，基於批內負樣本和難負樣本進行優化：

$$\min \mathcal{L} = -\log \frac{\varphi(h_q^{inst}, h_{t^+})}{\varphi(h_q^{inst}, h_{t^+}) + \sum_{t^- \in \mathcal{N}} \varphi(h_q^{inst}, h_{t^-})}$$

其中 $\mathcal{N}$ 為所有負樣本的集合，$\varphi(h_q, h_t)$ 為計算查詢 $q$ 與目標 $t$ 之間匹配分數的函數。本文採用溫度縮放的餘弦相似度函數，即 $\varphi(h_q, h_t) = \exp\left(\frac{1}{\tau}\cos(h_q, h_t)\right)$，其中 $\tau$ 為溫度參數。

**3.3 多模態資料格式化**

為在多樣化任務與模態上訓練統一嵌入模型，我們對所有查詢—目標對採用標準化格式。每個訓練樣本表示為一對 $(q, t^+)$，其中 $q$ 為查詢，$t^+$ 為正樣本目標。兩個部分均可為單張圖像、多張圖像、文字，或文字與圖像的交錯序列。

除原始輸入對外，我們引入指令 $inst$，用於指定查詢與目標之間的任務特定關係，幫助模型在異質任務間更好地理解上下文並提升泛化能力。我們將指令應用於原始查詢 $q$，生成指令條件化版本 $q^{inst}$：

$$q^{inst} = \text{[VISUAL TOKEN] Instruct: \{task instruction\}} \backslash n \text{ Query: \{q\}}$$

其中 $\{task\ instruction\}$ 為對嵌入任務的單句描述，例如「找到包含以下視覺內容的影片：」。

$\text{[VISUAL TOKEN]}$ 為前置的模態特定 token，用於指示視覺輸入為圖像或影片。例如，Qwen2-VL 使用 `<|image_pad|>` 表示圖像輸入，使用 `<|video_pad|>` 表示影片輸入。

類似地，我們可選擇性地對目標輸入 $t^+$ 應用簡單指令以引導表示學習，例如「理解所提供影片的內容：」：

$$t^+ = \text{[VISUAL TOKEN] \{target instruction\}}$$

這種統一格式使模型能夠一致地處理多模態輸入，同時利用指令信號提升跨任務與跨模態的泛化能力。

**3.4 資料採樣策略**

為支援在異質資料來源上進行有效的多任務訓練，我們設計了一個靈活且可擴展的資料採樣流程，包含兩個核心組成部分。

首先，我們依據預定義的採樣權重表執行即時批次混合。該權重表指定了從每個資料集採樣的相對概率，從而實現對不同任務類型的可控曝光。透過在訓練過程中動態抽取樣本，我們確保均衡覆蓋，防止模型對單一模態或領域產生過擬合。

其次，我們引入交錯子批次策略，以增強對比學習的難度與穩定性。具體而言，每個完整批次（如大小為 1024）被劃分為若干較小的子批次（如 8 個大小為 128 的子批次），每個子批次獨立採樣。與逐樣本獨立採樣相比，將相似樣本組合為子批次可提高子批次內部的同質性，從而增加對比判別的難度。與此同時，將多個子批次交錯組合可在完整批次中保持跨任務多樣性，避免完全同質批次（即來自單一來源的批次）中常見的訓練不穩定問題。該策略在樣本多樣性與結構一致性之間取得平衡，有助於實現更穩定、更魯棒的優化動態。

# Experiments

**4.1 實驗設置**

**4.1.1 訓練資料**

為使 VLM2Vec-V2 能在多樣化的多模態任務上有效訓練，我們整理了一個包含三大主要來源的訓練資料集：影片語言指令資料、視覺文件檢索，以及基於圖像的視覺任務。

首先，我們使用來自 LLaVA-Hound（Zhang et al., 2024a）的訓練資料，其中包含由 ChatGPT 生成的合成影片—字幕對和影片問答樣本。具體而言，我們使用了 30 萬個影片—字幕對和 24 萬個影片問答對。對於字幕資料，我們採用兩種格式：以字幕為查詢、影片為目標的影片檢索設置，或以影片為查詢從候選字幕中檢索最相關的文字描述。

其次，針對視覺文件檢索任務，我們納入了來自 ViDoRe（Faysse et al., 2024）和 VisRAG（Yu et al., 2024）的資料集，包括 ColPali 訓練集（11.8 萬條）、VisRAG 合成資料（23.9 萬條）和 VisRAG 領域內資料（12.3 萬條），為基於圖像的文件理解與檢索提供訓練樣本。

最後，我們納入了來自 MMEB-train（Jiang et al., 2024）的圖文資料集，以支援跨多種視覺理解任務的泛化能力，涵蓋問答、分類、檢索和視覺定位等任務，有助於提升所學嵌入表示在多項任務上的魯棒性。

---

**4.1.2 訓練設置**

我們以 Qwen2-VL 2B（Wang et al., 2024b）為骨幹網路訓練 VLM2Vec-V2，批次大小為 1,024，訓練步數為 2K/5K 步，交錯子批次大小為 64，損失溫度設為 0.02。為支援可擴展訓練，我們使用 GradCache（Gao et al., 2021）以實現大型全域批次，所有實驗均在 8 張 H100 GPU 上進行。在參數高效訓練方面，我們透過 PEFT 框架（Mangrulkar et al., 2022）應用 LoRA 微調，秩設為 16，縮放因子 $\alpha = 32$。訓練和評估期間均使用 8 幀均勻取樣的幀序列表示每部影片。

我們以 Hit@1 作為所有影片和圖像任務的主要評估指標，衡量正確目標排在首位的查詢比例。對於視覺文件任務，為與該領域的先前工作保持一致，我們報告 NDCG@5 指標。

---

**4.1.3 基線模型**

我們與多個 VLM 嵌入模型進行比較，包括 GME（Zhang et al., 2024b）、VLM2Vec（Jiang et al., 2024）和 LamRA（Liu et al., 2024b），這些模型大多主要在圖文對上訓練。儘管這些模型並非專為影片任務設計，但許多模型可透過將多個幀編碼為連續圖像的方式適配影片輸入。在影片評估中，GME 和 LamRA 使用單一中間幀，其餘模型使用 8 幀均勻取樣。

此外，為在各模態間提供公平比較，我們將 VLM2Vec-V2 與各模態的代表性專用模型進行評估。具體而言，我們納入了 ColPali（Faysse et al., 2024）（v1.3），這是一個採用後期互動匹配機制、專為文件檢索設計的模型。

**4.2 主要結果**

表 2 呈現了 VLM2Vec-V2 與多種基線模型在涵蓋圖像、影片和視覺文件任務的 78 個資料集上的全面比較，完整結果詳見附錄 A.4。VLM2Vec-V2 取得了最高的整體平均分（58.0），超越了多個強基線模型，包括同樣基於 Qwen2-VL 骨幹網路構建的 GME、LamRA 和 VLM2Vec，充分體現了我們統一訓練方法在不同模態和任務上實現強勁均衡性能的有效性。

在圖像任務上，VLM2Vec-V2 表現優異，以較大差距超越大多數基線模型，儘管模型規模僅為 2B，其性能卻與 VLM2Vec-7B 相當。在影片任務上，儘管影片訓練資料相對有限，模型仍取得了具有競爭力的表現。在視覺文件檢索方面，VLM2Vec-V2 超越了所有 VLM2Vec 變體，但仍落後於專為 VisDoc 任務優化的 ColPali。

**4.3 消融分析**

**4.3.1 跨模態泛化**

為評估不同模態來源對模型性能的影響，我們考慮三類訓練資料：基於圖像的資料（Image）、視覺文件資料（VisDoc）和影片資料（Video）。除在各單一模態上訓練模型外，我們還構建了結合兩種模態的資料集（Image+VisDoc、Image+Video）以及三種模態全部結合的資料集（Image+VisDoc+Video）。該設置使我們能夠系統分析各模態的貢獻，以及多模態組合對模型性能的影響。透過比較單模態與多模態資料訓練的模型，我們旨在評估多模態訓練如何影響泛化能力與任務效果。消融研究中未使用 ViDoRe-V2。

如表 3 所示，在單模態模型中，基於圖像資料的訓練取得了最高的平均性能。在兩模態組合中，Image+Video 略優於 Image+VisDoc，尤其在圖像基準測試上有顯著提升。值得注意的是，融合全部三種模態帶來了最佳的視覺文件任務性能和最高的整體分數，凸顯了全面訓練資料的優勢。

**4.3.2 資料採樣策略消融研究**

作為消融研究的一部分，我們探究了交錯子批次對模型在三種模態上性能的影響。當交錯子批次大小（IB）設為 0 時，批次中的所有樣本從不同來源隨機抽取，不進行分組。設為 64 表示大小為 1,024 的批次被劃分為大小為 64 的子批次，每個批次包含來自 16 個不同來源的資料。在另一極端，IB 設為 1,024 意味著整個批次來自單一來源，實際上禁用了交錯策略。該設置使我們能夠分析不同程度的來源混合如何影響訓練動態與跨模態泛化。

如表 4 所示，增大子批次大小持續提升了 VisDoc 和 Video 的性能。相反，圖像模態的最佳性能出現在子批次大小為 64 時，呈現倒 U 形趨勢——從 0 到 64 單調遞增，隨後從 64 到 1,024 單調下降。
